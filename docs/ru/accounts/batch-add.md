# Массовое добавление аккаунтов

Если нужно добавить сразу много аккаунтов, удобнее не заполнять форму на каждый вручную, а воспользоваться массовым добавлением.

Нажмите **Add profile** → **Batch add**.

<figure><img src="../.gitbook/assets/image (25).png" alt=""><figcaption></figcaption></figure>

## Подготовка файлов

Перед нажатием кнопки добавления нужно заранее подготовить три вещи рядом с папкой панели:

1.  **`config/logpass.txt`** — по одному аккаунту на строку, в формате:

    ```
    login:password
    ```
2.  **`config/proxy.txt`** — по одному прокси на строку, в формате:

    ```
    ip:port:login:password
    ```

    Количество строк и порядок в этом файле должны **точно совпадать** со строками в `logpass.txt` — первый прокси достанется первому аккаунту, второй — второму и так далее.
3. Папка **`maFile`** (лежит рядом с панелью, без вложенности в `config`) — в неё нужно положить файл авторизатора для каждого аккаунта, назвав файл **точно так же, как логин Steam** этого аккаунта (регистр не важен, расширение файла любое). Например, для аккаунта с логином `login123` файл должен называться `login123.maFile`.

## Запуск добавления

Когда все три файла на месте, нажмите кнопку **ADD** в окне массового добавления.

Панель построчно проверит соответствие логинов, прокси и maFile-файлов и покажет прогресс добавления с построчным логом: зелёным — успешно добавленные аккаунты, красным — строки с ошибкой (например: логин уже существует, нет подходящего maFile, несовпадение количества строк в файлах).

По завершении появится итоговое уведомление о том, сколько аккаунтов добавлено успешно и сколько строк не удалось обработать.

## Важно

Массовое добавление ничего не удаляет и не перезаписывает — оно только читает `logpass.txt`/`proxy.txt`/папку `maFile` и добавляет новые профили тем же способом, что и обычное ручное добавление. Уже добавленные ранее аккаунты не пострадают, даже если их логины всё ещё остаются в этих файлах.
